Hopping across the country with WireGuard
Feb 21, 2021
2 minutes read

I need to be able to access all the computers over the internet. That’s because my compute is spread across the country:

  • descartes: My dad’s Debian desktop (1800 km away).
  • kant: RPi 3 running Arch (a NAS, printer, a wormhole).
  • camus: RPi 4 running Manjaro (my personal work machine).
  • nietzsche: My wife’s Manjaro notebook.
  • Phones: newton, feyman, einstine and sagan.
  • kepler: a tiny Debian on Google Cloud.
  • webb: a tiny Ubuntu Server on Oracle Cloud.

All the computers actually have different astronomy inspired names.

Additionally, I found my lost RPi 2 hidden in a bag’s pocket, so it’ll also find a place among all the others (likely a second wormhole at my dad’s).

For a long time, I was using reverse SSH based tunnels, which would actually work reliably, but was slow (a US based VM). For the last year or so though, I have been using ZeroTier which works amazingly well. It being peer-to-peer, it is very low latency. But I cannot rely it to be forever around for me to use, hence the reverse SSH tunnel was a redundancy.

That is, until now. I finally got around to setup a WireGuard server providing me a VPN that I control. WireGuard is surprisingly simple to setup and works reliably. There are multiple guides on how to set it up (hence I didn’t yet another), like this one. The hop latency doesn’t take much of a hit because the cloud VM sits near the line connecting me and my parents.

I still use ZeroTier for most work, but I have a piece of mind that I’ll certainly have a way to service my dad’s computer if ZeroTier or GCP suddenly decide to vanish.

Interestingly, the VPNs aren’t even the primary network tool that I use. All these devices share most of their data with each other via syncthing. Syncthing is an awesome piece of technology allowing me to have redundancy and local access to files. Run a relay or donate if you can.
