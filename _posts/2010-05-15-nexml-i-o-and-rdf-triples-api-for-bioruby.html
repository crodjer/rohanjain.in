Pure fun. My job is to add NeXML parsing and serializing support, and an RDF triples API to BioRuby.

NeXML is a file format, based on XML, to express Phylogenetic data. Phylogenetics is the study of evolutionary relatedness among various groups of organisms. Evolution is a branching process wherein a new species evolves from the existing populace. Evolutionary relatedness between such species can be studied by DNA sequence analysis( and some other methods ). This can be visualized as a tree, the phylogenetic tree. Naturally, phylogenetic data will involve working with a set of species, a tree to depict their evolutionary relatedness and how such relations have been inferred.

RDF( or, Resource Description Framework ) is a meta data model. Conceptually, you use RDF to assert relation between, a subject and an object using a predicate. Say, the colour of my car is black. RDF breaks the previous assertion into (subject, predicate, object) => (car, color, black), also called triples. Phylogenetic database by nature is huge and complex. To find the relevant information efficiently, it is necessary to index phylogenetic data with descriptive information( or, meta data ).

The project has been offered under National Evolutionary Synthesis Center( NESCENT ). I will be mentored by Dr. Rutger Vos (primary), and Dr. Jan Aerts.

Coding starts from 24th May. Interested souls might find the "phylosoc" label on this blog and the project wiki[1] interesting.
