Is it just HTML/CSS? No. Add Javascript to that. Is that it? No. Aah, you missed out PHP. So, does that fill-in the gap. The answer still is, NO.

Quoting directly from Wikipedia:
Web development is a broad term for any activity related to developing a web site for the World Wide Web or an intranet. This can include e-commerce business development, web design, web content development, client-side/server-side scripting, and web server configuration. However, among web professionals, "web development" usually refers only to the non-design aspects of building web sites, e.g. writing markup and coding. Web development can range from developing the simplest static single page of plain text to the most complex web-based internet applications, electronic businesses, or social network services.

Here is what you need to understand and learn. Any website is made up of three parts:
  1. The Data Layer - All the data/information relevant to your site is ultimately stored in a Database. Say, for a stock exchange website, the data is the name of the companies, the current value of the stocks, the previous values of the stocks and so on.
  2. The User Interface Layer - This is your point of interaction with the user, where you collect data from him or present him with some data. For example you could display a form for registering users on your site or collecting some feedback, or, the stock exchange site showing you the results of your query.
  3. The Controller Layer - It is the pathway for your data to pass from the Data Layer to the User Interface layer and vice versa. This is where you perform queries to your database. Take the stock exchange site again for example. When you query for a company name say Reliance. So what you do is obtain the search term from the user (read passing the data from), query the database and display the result (read pass on the information to the user interface layer).
