--- name: seamlessly-use-github-behind-http-proxy-on-port-443 layout: post title: Seamlessly use Github behind http proxy on port 443 time: 2010-04-14 02:07:00.007000 +05:30 tags: - linux - debian - git - github - proxy --- This works if you proxy allows connection on port 443( used for https ). Open up ~/.ssh/config and enter the following lines( appropriately filling in values for proxy_host and proxy_port ):

Host gitproxy
User git
HostName ssh.github.com
Port 443
ProxyCommand corkscrew proxy_host proxy_port %h %p
IdentityFile /home/yeban/.ssh/id_rsa

Now, if your repository URL is git@github.com:username/test.git, you would use it as git@gitproxy:username/test.git

Courtsey: http://blog.codeslower.com/2008/8/Using-PuTTY-and-SSL-to-securely-access-GitHub-repositories-via-SSH

Update:
In the Host section, if you put github.com instead of gitproxy, then you can use your repository's git url unaltered.
